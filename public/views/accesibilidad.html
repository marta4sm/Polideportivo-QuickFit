Declaración de Accesibilidad

Situación de cumplimiento

El Polideportivo QuickFit se ha comprometido a hacer accesible su sitio web de conformidad con el Real Decreto 1112/2018, de 7 de septiembre, sobre accesibilidad de los sitios web y aplicaciones para dispositivos móviles del sector público.

Este sitio web es parcialmente conforme con el RD 1112/2018 y cumple los siguientes requisitos utilizando las técnicas WCAG 2.1 mostradas a continuación:

1.1.1 Contenido no textual - Nivel A

Todo el contenido que no es de texto que se presenta al usuario tiene una alternativa de texto que cumple un propósito equivalente.

  • G94: Proporcionar una alternativa de texto corto para contenido que no sea de texto que tenga el mismo propósito y presente la misma información que el contenido que no sea de texto
  • G95: Proporcionar alternativas de texto corto que brinden una breve descripción del contenido que no es de texto
  • G82: Proporcionar una alternativa de texto que identifique el propósito del contenido que no es de texto
  • ARIA6: Uso de aria-label para proporcionar etiquetas para objetos
  • ARIA10: Uso de aria-labelledby para proporcionar una alternativa de texto para contenido que no es de texto.
  • G196: Usar una alternativa de texto en un elemento dentro de un grupo de imágenes que describe todos los elementos del grupo.
  • H37: Usando atributos alt en elementos img

1.3.1 Información y relaciones - Nivel A

La información, la estructura y las relaciones transmitidas a través de la presentación se pueden determinar mediante programación o están disponibles en el texto.

  • Técnica G115: Uso de elementos semánticos para marcar la estructura
  • H49: Uso de marcado semántico para marcar texto especial o enfatizado
  • H48: Uso de ol, ul y dl para listas o grupos de enlaces
  • H42: Uso de h1-h6 para identificar encabezados
  • ARIA12: Usar role=heading para identificar encabezados
  • ARIA13: Uso de aria-labelledby para nombrar regiones y puntos de referencia

1.3.2 Secuencia significativa - Nivel A

Cuando la secuencia en la que se presenta el contenido afecta su significado, se puede determinar mediante programación una secuencia de lectura correcta.

  • G57: Ordenar el contenido en una secuencia significativa

1.3.4 Orientation - Level AA

El contenido no restringe su visualización y funcionamiento a una única orientación de visualización, como vertical u horizontal, a menos que sea esencial una orientación de visualización específica.

Usar CSS para establecer la orientación para permitir tanto una orientación vertical como horizontal.

1.3.5 Identify Input Purpose - Level AA

El propósito de cada campo de entrada que recopila información sobre el usuario se puede determinar mediante programación.

El contenido se implementa utilizando tecnologías con soporte para identificar el significado esperado para los datos de entrada del formulario.

  • H98: Uso de atributos de autocompletar de HTML 5.2

1.4.1 Use of Color - Level A

El color no se utiliza como el único medio visual para transmitir información, indicar una acción, provocar una respuesta o distinguir un elemento visual.

  • G14: Garantizar que la información transmitida por las diferencias de color también esté disponible en el texto
  • G205: Incluyendo una señal de texto para etiquetas de control de formulario de colores
  • G183: Usar una relación de contraste de 3:1 con el texto que lo rodea y proporcionar señales visuales adicionales al pasar el mouse por encima de los enlaces o controles donde solo se usa el color para identificarlos
  • C15: Uso de CSS para cambiar la presentación de un componente de la interfaz de usuario cuando recibe el foco

1.4.3 Contraste (Mínimo) - Nivel AA

Para texto de menos de 18 puntos si no está en negrita y de menos de 14 puntos si está en negrita:

  • G183: Usar una relación de contraste de 3:1 con el texto que lo rodea y proporcionar señales visuales adicionales al pasar el mouse por encima de los enlaces o controles donde solo se usa el color para identificarlos.

Para texto de al menos 18 puntos si no está en negrita y de al menos 14 puntos si está en negrita:

  • G145: Garantizar que exista una relación de contraste de al menos 3:1 entre el texto (y las imágenes del texto) y el fondo detrás del texto.

1.4.4 Cambio de tamaño del texto - Nivel AA

A excepción de los subtítulos y las imágenes de texto, el texto se puede cambiar de tamaño sin tecnología de asistencia hasta en un 200 por ciento sin pérdida de contenido o funcionalidad.

  • G142: Usar una tecnología que tiene agentes de usuario comúnmente disponibles que admiten zoom.

1.4.11 Contraste de no texto - Nivel AA

La presentación visual de los Componentes de Interfaz de Usuario y Objetos Gráficos tienen una relación de contraste de al menos 3:1 frente a los colores adyacentes.

  • G195: Uso de un indicador de enfoque altamente visible proporcionado por el autor.

1.4.12 Espaciado de texto - Nivel AA

En el contenido implementado, aplicando las siguientes reglas de estilo, no se pierde contenido ni funcionalidad.

  • Interlineado de al menos 1,5 veces el tamaño de fuente.
  • Espaciado de párrafos de al menos 2 veces el tamaño de la fuente.
  • Espaciado entre letras de al menos 0,12 veces el tamaño de la fuente.
  • Espaciado entre palabras de al menos 0,16 veces el tamaño de la fuente.

C36: Permitir anular el espaciado de texto.

1.4.13 Contenido al pasar el mouse o enfocar - Nivel AA

Cuando se recibe el foco o se apunta con el cursor a un elemento y esto desencadena que aparezca/desaparezca contenido adicional, este es persistente y además se puede mover el puntero sobre el contenido adicional sin que este desaparezca.

  • ARIA: Uso de role="tooltip".
  • CSS: Uso de pseudo clases de hover y focus.

2.1.1 Teclado - Nivel A

Toda la funcionalidad del contenido se puede operar a través de una interfaz de teclado sin requerir tiempos específicos para pulsaciones de teclas individuales.

  • G202: Garantizar el control del teclado para todas las funciones.
  • H91: Uso de enlaces y controles de formulario HTML.
  • G90: Proporcionar controladores de eventos activados por teclado.

2.1.2 Sin atrapamiento del teclado - Nivel A

Si el foco del teclado se puede mover a un componente de la página usando una interfaz de teclado, entonces el foco se puede alejar de ese componente usando solo una interfaz de teclado.

  • G21: Garantizar que los usuarios no queden atrapados en el contenido.

2.4.2 Título de página - Nivel A

Las páginas web tienen títulos que describen correctamente el tema o el propósito.

  • G88: Proporcionar títulos descriptivos para páginas web.
  • H25: Proporcionar un título utilizando el elemento de title.

2.4.3 Orden de enfoque - Nivel A

Se puede navegar secuencialmente por cada página web y los componentes enfocables reciben el enfoque en un orden que preserva el significado y la operabilidad.

  • G59: Colocar los elementos interactivos en un orden que sigue secuencias y relaciones dentro del contenido.
  • H4: Crear un orden de tabulación lógico a través de enlaces, controles de formulario y objetos.

2.4.4 Propósito del enlace (En contexto)

El propósito de cada enlace se puede determinar a partir del texto del enlace solo o del texto del enlace junto con su contexto de enlace determinado mediante programación.

  • G91: Proporcionar texto de enlace que describa el propósito de un enlace.
  • H24: Proporcionar alternativas de texto para los elementos de área de los mapas de imágenes.
  • ARIA8: Uso de aria-label para describir el fin del enlace.

2.4.5 Múltiples formas - Nivel AA

Existe más de una manera disponible para ubicar una página web dentro de un conjunto de páginas web.

  • G125: Proporcionar enlaces para navegar a páginas web relacionadas.
  • G63: Proporcionar un mapa del sitio.
  • G161: Proporcionar una función de búsqueda para ayudar a los usuarios a encontrar contenido.

2.4.6 Encabezados y etiquetas - Nivel AA

Los encabezados y las etiquetas describen el tema o el propósito.

  • G130: Proporcionar encabezados descriptivos.
  • G131: Proporcionar etiquetas descriptivas.

2.4.7 Enfoque visible - Nivel AA

Cualquier interfaz de usuario operable por teclado tiene un modo de operación donde el indicador de enfoque del teclado es visible.

  • G149: Uso de componentes de interfaz de usuario que son resaltados por el agente de usuario cuando reciben el foco.
  • C15: Uso de CSS para cambiar la presentación de un componente de la interfaz de usuario cuando recibe el foco.
  • G195: Uso de un indicador de enfoque altamente visible proporcionado por el autor.

3.1.1 Idioma de la página - Nivel A

El idioma humano predeterminado de cada página web se puede determinar mediante programación.

  • H57: Uso de atributos de idioma en el elemento html.

3.2.2 En entrada - Nivel A

Cambiar la configuración de cualquier componente de la interfaz de usuario no provoca automáticamente un cambio de contexto a menos que se haya informado al usuario sobre el comportamiento antes de usar el componente.

  • G80: Proporcionar un botón de envío (submit) para iniciar un cambio de contexto.
  • H32: Proporcionar botones de envío (submit).

3.2.3 Navegación consistente - Nivel AA

Los mecanismos de navegación que se repiten en varias páginas web dentro de un conjunto de páginas web ocurren en el mismo orden relativo cada vez que se repiten, a menos que el usuario inicie un cambio.

  • G61: Presentar componentes repetidos en el mismo orden relativo cada vez que aparecen.

3.2.4 Identificación coherente - Nivel AA

Los componentes que tienen la misma funcionalidad dentro de un conjunto de páginas web se identifican de forma coherente.

  • G197: Uso consistente de etiquetas, nombres y alternativas de texto para contenido que tiene la misma funcionalidad.

3.3.1 Identificación de errores - Nivel A

Si se detecta automáticamente un error de entrada, se identifica el elemento que tiene el error y se describe el error al usuario en texto.

  • G83: Proporcionar descripciones de texto para identificar los campos obligatorios que no se completaron.
  • G84: Proporcionar una descripción de texto cuando el usuario proporciona información que no está en la lista de valores permitidos.
  • G85: Proporcionar una descripción de texto cuando la entrada del usuario está fuera del formato o los valores requeridos.

3.3.2 Etiquetas o Instrucciones - Nivel A

Se proporcionan etiquetas o instrucciones cuando el contenido requiere la intervención del usuario.

  • G131: Proporcionar etiquetas descriptivas.
  • G89: Proporcionar formato de datos esperado y ejemplo.
  • G184: Proporcionar instrucciones de texto al comienzo de un formulario o conjunto de campos que describen la entrada necesaria.
  • G83: Proporcionar descripciones de texto para identificar los campos obligatorios que no se completaron.
  • H90: Indicación de controles de formulario requeridos mediante etiqueta o leyenda.
  • H44: Uso de elementos de etiqueta para asociar etiquetas de texto con controles de formulario.

3.3.3 Sugerencia de Errores - Nivel AA

Si se detecta automáticamente un error de entrada y se conocen sugerencias para su corrección, estas sugerencias se proporcionan al usuario.

  • G83: Proporcionar descripciones de texto para identificar los campos obligatorios que no se completaron.
  • G85: Proporcionar una descripción de texto cuando la entrada del usuario está fuera del formato o los valores requeridos.

4.1.1 Análisis de sintaxis - Nivel A

En el contenido implementado mediante lenguajes de marcado, los elementos tienen etiquetas de inicio y finalización completas, los elementos se anidan de acuerdo con sus especificaciones, los elementos no contienen atributos duplicados y cualquier ID es único.

  • G134: Validación de páginas web.
  • G192: Totalmente conforme a las especificaciones.
  • H88: Uso de HTML de acuerdo con las especificaciones.

Contenido no accesible

El contenido que se recoge a continuación no es accesible por lo siguiente:

  • Fallo en 3.1.2 Language of Parts - Level AA. El idioma humano de cada pasaje o frase en el contenido no se puede determinar mediante programación.
  • Fallo en 2.4.1 Saltar bloques - Nivel A. No existe un mecanismo disponible para eludir bloques de contenido que se repiten en varias páginas web.
  • En algunas páginas los headers no escalan secuencialmente de uno en uno
  • Podrían existir fallos puntuales de edición en alguna página web.

Atajos de teclado

  • Navegar hacia arriba: Ctrl + Shift + U
  • Navegar hacia abajo: Ctrl + Shift + W
  • (Activar / Desactivar) botón de navegación hacia arriba: Ctrl + Shift + X
